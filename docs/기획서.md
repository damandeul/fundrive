# 부릉부릉 — 서비스 기획서

> 지루한 운전을 재미있게, 재미있는 운전을 안전하게.
> 안전 운전 게임화 + 번호판 미니게임 + 드라이브 SNS를 결합한 모바일 웹앱

- 작성일: 2026-07-05
- 버전: v0.1 (MVP 기획)

---

## 1. 한 줄 정의

**"운전할수록 캐릭터가 자라는 안전운전 드라이브 앱"**

GPS로 규정속도 준수를 측정해 안전점수를 주고, 앞차 번호판으로 미니게임을 즐기고,
운전 중 겪은 에피소드를 인스타그램처럼 공유하는 올인원 드라이빙 컴패니언.

## 2. 문제 정의 & 기회

| 문제 | 부릉부릉의 해법 |
|---|---|
| 매일 반복되는 출퇴근 운전이 지루하다 | 번호판 미니게임, 스트릭, 주간 리그로 매 주행에 재미 부여 |
| 안전운전은 보상이 없어 동기가 약하다 | 안전점수 → 포인트 → 캐릭터 성장으로 즉각적 보상 |
| 운전 중 겪은 재미있는 순간을 나눌 곳이 없다 | 음성 메모로 기록하고 도착 후 피드에 공유 |

## 3. 타겟 사용자 (20문답 확정)

- **출퇴근 직장인** — 반복 주행의 지루함 해소가 핵심 니즈
- **초보 운전자** — 안전 습관을 게임처럼 형성
- **가족 단위 운전자** — 동승자(아이)와 함께하는 번호판 게임
- **장거리/직업 운전자** — 긴 주행 시간의 스트릭·리포트 활용

핵심 가치는 **안전 + 재미 + 커뮤니티의 균형**. 모든 주행에서 상시 사용 가능.

## 4. 핵심 기능

### 4.1 안전 주행 (GPS 안전점수)
- 주행 시작/종료 버튼 → 스마트폰 GPS로 속도·거리 추적 (별도 장비 불필요)
- 도로 제한속도 대비 초과 여부로 **실시간 안전점수** (100점 시작, 초과 시 감점)
- **감점만, 비난 없음** — 부정적 메시지 최소화로 이탈 방지 (확정 정책)
- 주행 종료 시 **주행 리포트 카드** 자동 생성 (거리·시간·평균/최고속도·점수)
- **무과속 스트릭**: 과속 없는 완주가 이어지면 연속 기록 보상
- 음성 코치: 과속 시 부드러운 음성 안내 (2차 버전)

### 4.2 번호판 미니게임 (동승자·정차 중심)
1. **암산 게임** — 앞차 번호판 4자리로 덧셈/목표수 만들기
2. **숫자 빙고/컬렉션** — 번호판에서 0~9 숫자를 모아 빙고 완성
3. **숫자 예언** — "다음 번호판에 7이 있다!" 예측 게임

### 4.3 드라이브 피드 (인스타그램형)
- 콘텐츠: 주행 리포트 카드 / 운전 에피소드 스토리 / 게임 성적·배지 자랑 / 드라이브 코스 추천
- 기록 방식: **음성 메모(주행 중)** + 도착 후 작성 + 원탭 북마크 + 동승자 기록
- 구조: 전체 공개 피드 + 팔로우 + 지역/코스 기반 + 안전점수 리더보드
- **프라이버시: 사진 업로드 시 타인 번호판·얼굴 자동 블러** (법적 필수)

### 4.4 보상 시스템
- **포인트 + 레벨**: 안전 주행·게임으로 포인트 적립, 레벨업
- **배지/칭호**: "1000km 무과속", "빙고 마스터" 등 업적 수집
- **캐릭터 성장**: 마스코트 '부릉이'가 안전 운전할수록 진화 (알 → 새싹카 → 스포츠카 → 전설의 드라이버)
- **주간 리그**: 비슷한 점수대끼리 묶어 승강제 경쟁

## 5. 안전·법규 정책

> ⚠️ 운전 중 휴대전화 조작은 도로교통법 위반 소지가 있음. 서비스 신뢰성을 위해 아래 정책 권장.

- 앱 시작 시 "운전자는 음성/자동 기능만, 화면 조작은 동승자·정차 중에만" 안내 문구 필수
- 게임 UI에 "동승자 모드" 표기
- 주행 리포트·피드 작성은 주행 종료 후로 유도
- 사진 공유 시 번호판·얼굴 자동 블러 (개인정보보호법 대응)
- 위치 정보는 주행 기록 목적으로만 수집, 명시적 동의 획득 (위치정보법 대응)

## 7. 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 프론트 | React + Vite + TypeScript + Tailwind CSS | 모바일 웹앱 우선 (확정) |
| 위치/속도 | 브라우저 Geolocation API | 스마트폰 GPS만 사용 (확정) |
| 백엔드/DB | Supabase (Auth + Postgres + Storage) | 확정. MVP는 로컬 저장 → 2단계 연동 |
| 배포 | Vercel | GitHub 연동 자동 배포 |
| 수익화 | 없음 (무료) | 사용자 확보 우선 (확정) |
